3. Quy tắc ưu tiên quyền sử dụng
3.1 Hai chế độ (quan trọng Phase 1)
Biến env	Hành vi
SAAS_ENFORCEMENT=0 (mặc định Phase 1)	Chỉ logic cũ: trial, api_access_enabled, credits (nếu bật). Bảng SaaS chỉ đọc/ghi thử.
SAAS_ENFORCEMENT=1 (Phase 2+)	Gọi entitlement_service trước khi chạy tính năng tốn phí.
Phase 1 implement service + rule nhưng mặc định enforcement tắt.

3.2 Thứ tự ưu tiên (khi SAAS_ENFORCEMENT=1)
Giải thích từng bước như “cầu thang” — bước trên pass thì xét tiếp; fail thì Deny (trừ bước cuối).

Bước 0 — Tài khoản có viết được không?
  role viewer / status suspended → DENY (giữ rbac hiện tại)
Bước 1 — ADMIN (role = admin)?
  YES → ALLOW mọi feature + quota (không trừ monthly_usage trừ khi bật tracking admin)
  Lưu ý LLM: admin vẫn dùng pool/key như `user_api_access` hiện tại
Bước 2 — SUBSCRIPTION active (trialing/active, còn period)?
  YES → Lấy plan → đọc usage_limits[feature_key]
       → So monthly_usage với limit_value
       → Còn quota → ALLOW (và sau khi success mới record event)
       → Hết quota → xuống Bước 5 (Deny) hoặc gợi ý nâng cấp
  NO  → Bước 3
Bước 3 — API access enabled (api_access_enabled = true)?
  YES → ALLOW theo logic CŨ (bypass trial như hiện tại)
       Phase 2: có thể vẫn áp quota “mềm” nếu muốn
  NO  → Bước 4
Bước 4 — TRIAL còn hạn (user_trials.ends_at > now)?
  YES → ALLOW (giữ require_active_trial hiện tại)
       Tùy chọn sau: map trial → plan slug free_trial + check usage_limits
  NO  → Bước 4b
Bước 4b — CREDIT (CREDITS_ENABLED=1 và credit_balance đủ)?
  YES → ALLOW + consume_credits() như credits.py
  NO  → Bước 5
Bước 5 — DENY
  HTTP 403 / 402 với message rõ: hết trial, chưa gói, hết quota, liên hệ admin
3.3 Bảng tóm tắt (theo yêu cầu của bạn)
Thứ tự	Điều kiện	Kết quả
1	Admin	Cho phép (full)
2	Subscription active + còn quota feature	Cho phép
3	API access enabled	Cho phép (legacy; Phase 1 = như hiện tại)
4	Trial còn hạn	Cho phép (legacy)
5	Credit đủ (nếu hệ thống credit bật)	Cho phép + trừ credit
6	Không đủ	Deny
Tách biệt hai khái niệm (tránh nhầm):

Khái niệm	Trả lời câu hỏi
Entitlement (được phép dùng tính năng?)	Subscription / trial / admin / API flag
Quota (còn lượt trong tháng?)	usage_limits + monthly_usage
API / LLM (gọi OpenAI được không?)	api_access_enabled + key — giữ assert_user_may_use_api
Subscription active không tự thay API access — user trả gói vẫn cần key riêng hoặc admin bật API Admin pool (trừ khi sau này gói Pro bao gồm pool).
