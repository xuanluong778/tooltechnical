# User Dashboard & Onboarding

Tài liệu này mô tả dashboard SaaS cho user và luồng onboarding cho người dùng mới.

---

## 1. Vì sao cần dashboard SaaS

Hiện tại /report mạnh về Technical SEO, nhưng SaaS cần một dashboard tổng quan cho user sau khi đăng nhập.

Đề xuất route:

```txt
/dashboard
hoặc
/app
```

---

## 2. Thành phần dashboard

```txt
Gói hiện tại
Credit/quota còn lại
Số bài Content AI đã tạo trong tháng
Số lần Technical Audit đã chạy trong tháng
Số keyword research đã dùng
Website WordPress đã kết nối
Hoạt động gần đây
Cảnh báo gần hết quota
Nút nâng cấp gói
```

---

## 3. Quick Actions

```txt
Quét website
Tạo bài viết SEO
Research keyword
Kết nối WordPress
Tạo schema
Xem báo cáo gần nhất
```

---

## 4. Empty state cho user mới

```txt
Bạn chưa có dữ liệu nào.
Hãy bắt đầu bằng cách quét website đầu tiên hoặc tạo bài viết SEO đầu tiên.
```

CTA nên có:

```txt
Quét website đầu tiên
Tạo bài SEO đầu tiên
Kết nối WordPress
```

---

## 5. Luồng onboarding

Sau đăng ký, user nên đi qua onboarding.

```txt
Bước 1: Bạn muốn dùng BeeSEO để làm gì?
- Audit Technical SEO
- Viết content SEO
- Research keyword
- Đăng bài WordPress
- Làm SEO cho khách hàng

Bước 2: Bạn có website chưa?
- Có
- Chưa

Bước 3: Nhập domain website

Bước 4: Có muốn kết nối WordPress không?

Bước 5: Gợi ý bước tiếp theo
```

---

## 6. Sau onboarding, dashboard gợi ý

```txt
1. Quét website đầu tiên
2. Research bộ từ khóa đầu tiên
3. Tạo bài SEO đầu tiên
4. Kết nối WordPress
```
